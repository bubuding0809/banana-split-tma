Service health overview

Day-one Axiom board for operational triage and morning glance

Design spec · 2026-05-12 · bananasplitz dataset

Why this board

Today

  • One Axiom dataset: bananasplitz
  • One dashboard: Group reminder reconciliation
  • Covers the weekly self-heal cron — nothing else

Gap

  • No bot health view
  • No tRPC error / latency view
  • No "is anything on fire" glance

Goal: one board that reads cleanly at any time window — 15m for triage, 24h for morning, 7d for trend.

Surfaces (the grouping axis)

Every chart derives a surface column. Four surfaces, evaluated in order:

MatchSurface
service == "bot"bot
service == "lambda" AND action startswith "agent."agent
service == "lambda" AND (procedure != "" OR action == "trpc.call.start")trpc
everything elselambda (catch-all)

After live, spot-check the catch-all distribution and refine if needed.

Catches both failure modes: error spikes (rate) and silent failures where traffic drops to zero (delta).

Reused APL: surface derivation

| extend surface = case( service == "bot", "bot", service == "lambda" and action startswith "agent.", "agent", service == "lambda" and (isnotempty(procedure) or action == "trpc.call.start"), "trpc", "lambda" )

Repeated across 7 charts. Acceptable day-one duplication; can promote to a saved query later.

Headline pill query

['bananasplitz'] | extend surface = case(...) | where surface == "bot" | summarize total = count(), errors = countif(level >= 50) | extend rate = iff(total == 0, 0.0, 100.0 * errors / total) | project rate

Four copies of this, one per surface. Threshold rules on the Statistic chart drive color.

Time-series queries

Volume

['bananasplitz'] | extend surface = case(...) | summarize count() by bin_auto(_time), surface

Error rate %

['bananasplitz'] | extend surface = case(...) | summarize total = count(), errors = countif(level >= 50) by bin_auto(_time), surface | extend rate = iff(total == 0, 0.0, 100.0 * errors / total) | project _time, surface, rate

Top-N breakdowns

Top erroring actions

['bananasplitz'] | where level >= 50 and isnotempty(action) | summarize count() by action | top 10 by count_

Lambda action-prefix over time

['bananasplitz'] | where service == "lambda" and isnotempty(action) | extend prefix = case( action startswith "agent.", "agent", action startswith "expense.", "expense", action startswith "group.", "group", action startswith "reconciliation.", "reconciliation", "other" ) | summarize count() by bin_auto(_time), prefix

Fatal log tail

['bananasplitz'] | extend surface = case(...) | where level >= 50 | project _time, surface, action, ['err.message'], chat_id, request_id, ['err.code'] | sort by _time desc | take 100

LogStream chart. Click any row to jump into Axiom's explorer with the request_id pre-filtered.

Edge cases

Empty window

iff(total == 0, 0, …) in every Statistic — no NaN ever.

No prior traffic

Delta displays —, not Infinity.

Surface miscategorization

lambda is the catch-all. Probe its action distribution post-launch.

Time-window flexibility

bin_auto on every time-series — reads at 15m, 24h, and 7d.

Testing

  • Open at default 24h, confirm all panels render
  • Per-surface: explorer query matches pill exactly
  • Sum of surface volumes == total event count (no double-count)
  • Re-render cleanly at 15m and 7d
  • Pick a known error window, confirm pill + tail + top-actions all reflect it
