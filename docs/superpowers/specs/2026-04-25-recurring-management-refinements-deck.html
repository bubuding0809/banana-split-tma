Recurring Expense
Management — UX Refinements

Five small UX gaps in the recurring-expense feature shipped via PR #236 → #249. Each is small on its own; bundled they form a coherent "make recurring management actually findable and consistent" pass — most fixes are frontend-only, with one read-side Prisma include on the backend.

Status: Awaiting user review Author: bubu (with Claude Opus 4.7) Threads: 5 New routes: 0 New mutations: 0
In one screen

Five small fixes, one bundled spec

Thread 1

Surface the entry point

Move "Recurring expenses · Manage" from buried in Settings to the top of the Transactions tab — where users already think about recurring transactions.

Thread 2

Relocate Convert currencies

The freed-up slot wasn't lost — Convert currencies moves to the top of the Balances tab, its more natural home. FX is a balance concern, not a list concern.

Thread 3

Status-aware schedule rendering

Past expenses still show a canceled or ended template's schedule details, implying the schedule is live. Gate both the modal Schedule section and the ↻ badge on ACTIVE.

Thread 4

Manage shortcut from expense modal

From a recurring expense's modal, no path to edit the schedule. Add a one-tap "Manage schedule" affordance in the Schedule section.

Thread 5

Kill the silent no-op

The Repeat / End Date cells in the edit-expense form look editable but their values are never sent. Hide them in edit mode — recurrence stays a create-only concern.

Thread 3 · Status-aware rendering

The bug: canceled schedules keep claiming they're live

A canceled template still has a row in the DB (soft-delete sets status = "CANCELED"). The expense modal queries recurring.get, which doesn't filter by status — so the Schedule section keeps rendering as if the schedule is live. Same root cause for the ↻ badge on cell rows.

Status Modal Schedule section Cell ↻ badge Manage shortcut
ACTIVE Full section: Repeat + End Date Visible (link-blue, "Active recurring schedule") Yes (Thread 4)
ENDED Muted Caption: ↻ From a recurring schedule that ended on … Hidden No
CANCELED Hidden — modal looks like a one-off expense Hidden No

Asymmetry rationale: ENDED is a natural lifecycle outcome — a one-line breadcrumb is useful for understanding past spending. CANCELED is a deliberate user action — persisting its details on every materialized expense forever is noise.

Thread 4 · Direct path to schedule editor

Tap "Manage schedule ›" — skip the 4-tap detour

Before

  1. Close the expense modal
  2. Navigate to Settings → Recurring expenses · Manage
  3. Land on templates list
  4. Find the right row → tap → modal
  5. Tap pencil → schedule editor

5 taps + 2 context switches.

After

  1. From the expense modal, tap Manage schedule ›
  2. Land directly on /chat/$chatId/edit-recurring/$templateId

Single tap. Only renders for ACTIVE templates — no shortcut to edit a canceled or ended schedule (nothing to edit).

Bonus: ↻ badge semantics nailed down

Today the ↻ badge on ChatExpenseCell renders whenever recurringTemplateId is set — accidentally a "was recurring" historical marker. After this change, the badge means "is recurring (live state)" — only present when the linked template is ACTIVE. Tooltip updated to "Active recurring schedule" so the meaning is explicit.

Thread 5 · Edit-expense form

Stop pretending you can edit recurrence here

Bug today (silent no-op)

  • RepeatAndEndDateSection renders unconditionally in the form
  • Edit-expense form initializes recurrence to empty default — not the linked template's actual schedule
  • Submit handler never reads value.recurrence
  • Backend updateExpense doesn't even accept a recurrence field
  • Net: user toggles Repeat → picks Monthly → saves → nothing changes

Fix: hide cells in edit mode

  • AmountFormStep takes a mode: "add" | "edit" prop
  • The form gates RepeatAndEndDateSection on mode === "add"
  • Edit-expense path hides the section entirely — no affordance, no pretense
  • Schedule edits live exclusively at /chat/$chatId/edit-recurring/$templateId
  • Drop the now-unused recurrence: formOpts.defaultValues.recurrence line in EditExpensePage

Why hide rather than show a read-only indicator: ExpenseDetailsModal's Schedule section + Manage shortcut (Thread 4) is already the one canonical "is this expense recurring? where do I edit?" surface. A second indicator inside the edit form duplicates that without adding capability.

Verification

Tests + manual UAT

Component smoke

  • New entry cell: hidden on pending / empty list, renders with correct count, navigates on click
  • RecurringScheduleSection: full section for ACTIVE, muted Caption for ENDED, null for CANCELED. Manage tap navigates correctly.
  • ChatExpenseCell: badge only when recurringTemplate?.status === "ACTIVE"
  • ConvertCurrenciesCell: null when no foreign currencies; trigger + modal otherwise
  • AmountFormStep: Repeat section in "add", hidden in "edit"

Backend

getExpenseByChat returns recurringTemplate: { status } for expenses with a linked template, null otherwise. Existing callsites continue to work — additive change.

Manual UAT (full-environment lifecycle)

Single cycle covering each of the five threads, per the full-environment-UAT loop preference. After each: verify DB state via Supabase MCP and AWS Scheduler state via aws scheduler get-schedule.

  • Active templates → entry cell visible with correct count
  • Cancel a template → cell badge + modal Schedule section both disappear; Settings shows no duplicate entry
  • ACTIVE template → tap "Manage schedule ›" → lands on edit page with form populated
  • ENDED template → muted Caption renders; no Schedule section, no Manage tap
  • Edit existing expense → no Repeat / End Date cells. Add expense → cells visible, behave normally
  • Foreign-currency expenses → Convert cell appears at top of Balances; modal triggers; conversion completes
