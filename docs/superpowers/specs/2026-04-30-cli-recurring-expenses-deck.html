CLI Recurring Expenses

Achieving API Parity in the Command Line Interface

The Goal & Context

Current State: The backend tRPC API fully supports recurring expenses (creation, listing, updating, canceling) via the recurringRouter and AWS EventBridge scheduling.

The Gap: The CLI (`apps/cli`) currently only supports one-off expenses.

The Objective: Expose full recurring expense capabilities in the CLI to enable power-user workflows and automated agent integrations.

Tech Innovation Theme Slide 2

Architectural Approach

We are implementing the Distinct Namespace Approach:

  • Creation: Handled by enhancing the existing create-expense command with new flags. (Because creating a recurrence materializes the initial expense immediately).
  • Management: Handled by a brand new suite of isolated commands in a dedicated recurring namespace.

This approach keeps command flags tidy and mirrors the backend's strict separation between standard endpoints and the recurringRouter.

Tech Innovation Theme Slide 3

Enhancing create-expense

We will add five new optional flags to the existing command:

  • --recurrence-frequency: DAILY, WEEKLY, MONTHLY, YEARLY
  • --recurrence-interval: Default 1
  • --recurrence-weekdays: Comma-separated (e.g. MON,WED)
  • --recurrence-end-date: ISO 8601 string
  • --recurrence-timezone: Auto-defaults to system timezone
$ banana create-expense --amount 50 --description "Netflix" \
  --payer-id 123 --split-mode EQUAL --participant-ids 123,456 \
  --recurrence-frequency MONTHLY
Tech Innovation Theme Slide 4

New Management Commands

A new recurring.ts module will introduce 4 targeted commands:

  • list-recurring-expenses: Lists active templates for a chat context.
  • get-recurring-expense: Fetches full details for a given --template-id.
  • update-recurring-expense: Modifies schedule or amount/description on an active template.
  • cancel-recurring-expense: Soft-deletes/halts an active recurring template.
