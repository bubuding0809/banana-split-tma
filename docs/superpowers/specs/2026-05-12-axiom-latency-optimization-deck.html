Headline leaderboard query

['bananasplitz'] | where isnotnull(duration_ms) and isnotempty(msg) | summarize count = count(), p50_ms = percentile(duration_ms, 50), p95_ms = percentile(duration_ms, 95), p99_ms = percentile(duration_ms, 99) by msg | where count >= 5 | top 10 by p95_ms desc

Twin panel sorts by p99_ms desc instead — surfaces tails the p95 panel hides.

HTTP / tRPC drill-downs

By route

['bananasplitz'] | where msg == "req.end" and isnotnull(duration_ms) and isnotempty(path) | summarize count = count(), p50_ms = percentile(duration_ms, 50), p95_ms = percentile(duration_ms, 95), p99_ms = percentile(duration_ms, 99) by path | where count >= 5 | top 10 by p95_ms desc

By tRPC procedure

['bananasplitz'] | where msg == "trpc.call.end" and isnotnull(duration_ms) and isnotempty(procedure) | summarize count = count(), p50_ms = percentile(duration_ms, 50), p95_ms = percentile(duration_ms, 95), p99_ms = percentile(duration_ms, 99) by procedure | where count >= 5 | top 10 by p95_ms desc

Agent internals

Tool calls

['bananasplitz'] | where msg == "agent.tool.end" and isnotnull(duration_ms) and isnotempty(tool) | summarize count = count(), p50_ms = percentile(duration_ms, 50), p95_ms = percentile(duration_ms, 95), p99_ms = percentile(duration_ms, 99) by tool | top 10 by p95_ms desc

No count >= 5 filter — agent-tool data is sparse. Read with volume in mind.

LLM TTFB

['bananasplitz'] | where msg == "agent.llm.first_chunk" and isnotnull(latency_ms) | summarize p95_ms = percentile(latency_ms, 95)

p95 over time

['bananasplitz'] | where isnotnull(duration_ms) and msg in ( "<msg-1>", "<msg-2>", "<msg-3>", "<msg-4>", "<msg-5>" ) | summarize p95_ms = percentile(duration_ms, 95) by bin_auto(_time), msg

APL has no clean "top-N then time-series" pattern. The top-5 msg list is hard-coded at build time from a 7-day inspection. ~30-second manual refresh every couple weeks if rankings shift.

Edge cases

Sparse windows

Leaderboards filter count >= 5. At 15m window, panels may be near-empty — that's correct, not a bug.

Batched tRPC paths

Comma-joined paths bucket separately. Different perf profile — correct grouping.

Missing procedure tag

trpc.internal.timing has empty procedure on 428 events — excluded from drill-down. Future fix to the log line.

Agent tools tiny

6 events / 7d. Directional only until traffic grows.
